# MSA_TEMPLATE — 전체 구조 개요

재사용 가능한 **MSA 레퍼런스 템플릿**. "프론트 → 인증서버 → 백엔드"가 표준 패턴으로 동작하고, 서비스를 하나 더 붙일 때 무엇을 복제하면 되는지가 코드로 드러나는 것이 목표.

## 토폴로지

```
myFront(:5173)
   │  (로그인은 Keycloak로 리다이렉트)
   ▼
auth-server(:9000)  ──OIDC──▶  Keycloak(:8080)  ──▶ Google IdP
   │  자체 RS256 JWT 발급 + JWKS 공개 + /api/me + 백채널 로그아웃
   │
   ├──(Bearer AT)──▶ board-service(:9100)   JWT를 JWKS로 자체검증(SSO)
   └──(미래) gateway-server(:8000) 단일 진입점
                         │
   postgres(:5433): authdb / boarddb / keycloak
```

## 핵심 원칙 (이 설계를 관통하는 결정)

| # | 결정 | 근거 |
|---|---|---|
| 1 | 로그인 = **Keycloak 리다이렉트(OIDC)** + Google. ROPC/자체폼 안 씀 | 표준·SSO. UX 불만은 테마로 해결 |
| 2 | auth-server가 **자체 RS256 JWT** 발급(Keycloak 토큰을 그대로 안 줌) = BFF 패턴 | 프론트 계약을 우리가 통제 |
| 3 | **토큰 검증은 각 리소스 서버가 JWKS로 자체 수행** | 분산 검증. 발급자 1개(:9000), 신뢰는 JWKS로 |
| 4 | 로그아웃 = **백채널**(서버-서버 end_session, KC refresh_token 사용) | id_token_hint 방식은 만료 버그 → [[01 인증 플랫폼 (Keycloak BFF + 자체 JWT)]] |
| 5 | 각 서비스 = **별도 repo**. 우산(infra-settings)은 인프라+구조만 | [[06 저장소 구조 + 운영 원칙]] |
| 6 | 설계/작업 문서는 **Obsidian에만**, repo docs/ 금지 | 〃 |
