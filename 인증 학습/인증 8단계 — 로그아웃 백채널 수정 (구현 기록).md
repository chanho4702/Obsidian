# 인증 8단계 — 로그아웃 백채널 수정 (구현 기록)

> [[인증 8단계 — Keycloak BFF 자체-JWT 플랫폼 (구현 기록)]]의 후속 버그픽스.
> 그 노트의 로그아웃 흐름("end_session URL 반환 → 프론트 전체 페이지 이동")은 **이 노트로 대체됨.**
> 관련: [[인증 4단계. 토큰 보안 (Access·Refresh)]](RT 회전), [[인증 로드맵]](11단계 Single Logout).

## 한 줄

로그아웃을 눌러도 **Keycloak SSO 세션이 안 끊겨** 재로그인이 비밀번호 없이 즉시 됐다. 원인은 **금방 만료되는 id_token**에 로그아웃을 의존한 것. **서버가 Keycloak에 직접 로그아웃을 요청(백채널)** 하도록 바꿔 해결했다.

## 배경 — 토큰 3종

| 토큰 | 발급 | 수명 | 저장 | 용도 |
|---|---|---|---|---|
| 자체 AT | auth-server | 15분 | 프론트 메모리 | 보호 API 호출 |
| 자체 RT | auth-server | 14일 | HttpOnly 쿠키 | AT 재발급 |
| KC id_token / **refresh_token** | Keycloak | id_token은 수 분 | auth-server DB | KC 세션 식별/종료 |

- **SSO 세션**: Keycloak이 "이 사용자 로그인 중"이라 기억하는 것. 살아있으면 재로그인 시 비번을 안 물어본다.
- **로그아웃 = ① 우리 세션 정리 + ② KC SSO 세션 종료.** ②가 빠지면 "로그아웃했는데 재로그인 즉시"가 난다.

## 근본 원인 — 옛 방식(프론트채널 id_token_hint)

```
[로그인]  KC id_token 을 DB 저장
[로그아웃] 그 id_token 을 꺼내 브라우저를 KC end_session 으로 리다이렉트(증거로 첨부)
```

- **id_token 은 수 분이면 만료**되는데, 앱은 자체 RT(14일)로 계속 로그인 유지 → 로그아웃 시점엔 첨부할 증거가 이미 만료 → KC가 세션을 안 끊음.
- 게다가 프론트는 로그아웃 URL이 null이면 **조용히 로컬만 정리**하고 끝 → 겉보기 로그아웃, KC는 그대로.

> 비유: 호텔 체크아웃을 "(만료된) 임시 출입증을 보여줘야만 방 비워주는" 구조로 만든 셈. 출입증 만료 → 방(KC 세션) 계속 잡힘.

## 해결 — 백채널(서버-서버) 로그아웃

`platform-bff`는 client_secret을 가진 신뢰된 클라이언트라, 브라우저를 거칠 필요 없이 **서버가 KC에 직접 호출**할 수 있다. 만료 빠른 id_token 대신 **KC refresh_token**(세션과 함께 삶)을 쓴다.

```
[로그인] LoginSuccessHandler 가 KC refresh_token 을 DB(kc_refresh_token)에 저장 (rotate 시 승계)
[로그아웃 클릭]
  myFront → POST /api/auth/logout         (브라우저 리다이렉트 없음)
  auth-server:
    1) 자체 RT 패밀리 폐기 + RT 쿠키 삭제
    2) KeycloakLogoutClient → POST {issuer}/protocol/openid-connect/logout
         body: client_id, client_secret, refresh_token   → KC SSO 종료
  myFront → 메모리 AT 비움 → ProtectedRoute 가 /login 으로
```

> 비유: 이제 호텔 매니저(서버)가 하우스키핑(KC)에 **직접 전화**해 방을 비운다. 출입증 만료 무관.

장점: id_token 만료 무관 / 브라우저 :8080 왕복 불필요 / KC 세션 확실히 종료.

> ⚠️ **Google 로그인**은 KC 세션은 끊겨도 **Google 자체 세션**이 남아 재로그인이 빠를 수 있다 — 소셜 로그인의 정상 동작. 우리가 통제하는 범위는 KC 세션까지.

## 같이 잡은 별개 버그 — `@Param` 누락

로그아웃을 실제로 호출하자 런타임에서 터짐:
```
For queries with named parameters you need to provide names for method parameters; Use @Param ... or -parameters
```
- 원인: `RefreshTokenRepository.revokeFamily`의 `@Query(":familyId")`가 이름 기반인데, **실행 앱이 `-parameters` 없이 컴파일**돼 파라미터 이름이 `.class`에 안 남음.
- **테스트는 통과/런타임만 터짐** = gradle 테스트 빌드엔 이름 정보가 있었고 앱 빌드(IDE)엔 없던 불일치. 원래부터 잠복(재사용 탐지에도 쓰임)했고 로그아웃 호출로 처음 드러남.
- 수정: ① `@Param("familyId")` 명시(직접 해결) ② `build.gradle`에 `-parameters` 추가(테스트=런타임 일치, 재발 차단).

## 배운 점

- **로그아웃은 토큰 정리가 아니라 "세션 종료"가 본질.** 우리 쪽 토큰만 지우면 IdP 세션이 남아 반쪽짜리 로그아웃이 된다 → [[인증 로드맵]]의 11단계 Single Logout과 직결.
- **만료가 빠른 토큰을, 오래 사는 세션의 종료 증거로 쓰면 안 된다.** confidential 클라이언트면 refresh_token 기반 **백채널 로그아웃**이 정석.
- **테스트 green ≠ 런타임 안전.** 컴파일 플래그(`-parameters`) 같은 환경 차이가 잠복 버그를 숨긴다.
- 이 노트가 이 작업의 정식 기록이다(상세 문서를 repo에 따로 두지 않음 — 작업기록은 Obsidian에만).
