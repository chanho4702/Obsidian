# 인증 6+7단계 — 프론트엔드 완성 (회원가입·모듈화·UX)

> [[인증 6-7단계 OAuth2-OIDC 로그인]] 의 후속. 백엔드(OAuth2+OIDC+자체 JWT)는 이미 완성돼 있었고,
> 이번엔 **프론트(`C:\MSA_TEMPLATE\myFront`)의 회원가입을 채우고, 인증 전체를 재사용 모듈로 재구성**하고,
> 실사용에서 나온 UX 버그들을 잡았다.
> repo: 프론트 `chanho4702/ch-stack-frontend`(main) · 백엔드 `chanho4702/Oauth-OIDC-Login`(master)

## 핵심 한 줄
로그인/로그아웃은 이미 됐고 **회원가입만 비어 있었다.** 그걸 채우면서 흩어진 인증 코드를
**폴더째 복사 가능한 재사용 모듈**(`src/auth`, `src/notifications`)로 정리하고, 흰 화면 깜빡임·죽은 로그아웃 버튼 같은 UX 결함을 제거했다.

---

## 1. 한 일

### 회원가입 (없던 기능)
- 백엔드 `POST /api/auth/signup`(`{username,password}`, 중복 시 409)은 **이미 있었음** → 프론트만 비어 있었다.
- `client.signup(email, password)` → 가입 성공 후 **곧바로 `login()`까지(자동 로그인)**.
- `SignUpPage`(`/register`): 이메일 + 비밀번호 + 비밀번호 확인. 성공 시 `/app` 진입, 409면 "이미 가입된 이메일" 에러.
- ⚠️ 함정: `LoginPage`의 "회원가입" 링크가 `/sign-up`(MUI **참고용** 템플릿)으로 잘못 가 있었음 → 실제 `/register`로 수정.
  현 컨벤션: 실제 페이지 = `/login`·`/register`, 참고 템플릿 = `/sign-in`·`/sign-up`(손대지 않음).

### 재사용 모듈화 (핵심 요청)
- 흩어진 `src/lib/api.ts` + `src/app/auth/*` → **`src/auth/` 한 폴더로 통합**.
- 모듈 전역 싱글톤을 **`createAuthClient({ baseUrl })` 팩토리**로 전환 → 인스턴스마다 자기 AT·refresh dedup 보유.
  → **다른 프로젝트는 폴더째 복사 + `VITE_API_BASE`만 바꾸면 동작.** import 는 항상 배럴(`src/auth`)에서.
- `src/notifications/`: `NotificationProvider` + `useNotify()`(success/error/info/warning) 큐형 MUI Snackbar/Alert 토스트.

### UX / 에러 처리
- `RouteErrorPage`(모든 라우트 errorElement) + `NotFoundPage`(catch-all 404).
- `GuestRoute`(ProtectedRoute의 반대): 로그인 상태로 `/login`·`/register` 오면 `/app`으로.
- 로그인·회원가입·OAuth 실패 시 토스트 알림.

## 2. 디버깅 3건 (← 진짜 학습 포인트)

### (a) 좌측하단 로그아웃만 안 됨
- **증상:** 우측상단 로그아웃 버튼은 되는데 좌측하단 "계정 메뉴" 로그아웃은 무반응.
- **근본원인:** 좌측하단은 참고용 템플릿 `OptionsMenu`였고, 그 `Logout` 항목이 `onClick={handleClose}`만 호출 —
  **우리 `logout()`에 배선이 안 된 죽은 버튼**(Profile·Settings도 전부 placeholder).
- **교훈:** MUI 템플릿을 그대로 가져다 쓰면 그 안의 버튼은 *장식*이다. 실제 동작은 직접 배선해야 한다.
  백엔드 로그아웃 자체는 curl로 검증해 멀쩡했음(200 → 쿠키 삭제 → 이후 refresh 401 REVOKED). "안 되는 곳"부터 정확히 좁히는 게 핵심.
- **수정:** 참고 템플릿 수정 금지 규칙대로, 앱 버전 `AppOptionsMenu`를 만들어 실제 `onLogout`에 배선.

### (b) 세션 복원 중 흰 화면 번쩍임
- **증상:** `/app` 직접 진입/새로고침 때 흰(또는 빈) 화면이 잠깐 번쩍.
- **근본원인:** 이 코드베이스는 **페이지마다 각자 `AppTheme`로 감싸는데**, `ProtectedRoute`/`GuestRoute`의
  **로딩 상태만 테마 없는 맨 `<Box>`**였다. 세션 복원(`/refresh`+`/api/me` 네트워크 왕복) 동안 테마 없는 흰 화면이 노출됨.
- **수정:** 가드에 `fallback` prop 추가(모듈 portability 유지) → 앱에서 테마 입힌 `AuthLoadingScreen` 주입.

### (c) OAuth 로그인 후 콜백 페이지 깜빡임
- **증상:** Google 로그인 직후 중간 페이지가 한 번 번쩍이고 대시보드로 감.
- **근본원인:** 백엔드가 `/oauth/callback`으로 보내고 → 거기서 refresh 후 `/app`으로 **또** 이동하는 **2단계 전환**.
- **수정:** **콜백 페이지를 통째로 제거.** 백엔드 `frontend-redirect-uri`를 `/oauth/callback` → **`/app`**으로 바꾸니,
  프론트 `AuthProvider`가 마운트 시 하는 **silent refresh**가 세션을 복원 → 별도 콜백 페이지 불필요.
- **결정 메모(왜 `/login` 아니라 `/app`?):** 그 시점엔 이미 인증된 사용자다. `/login`으로 보내면 GuestRoute가
  다시 `/app`으로 redirect하는 **추가 홉**이 생긴다(= 방금 없앤 콜백 2단계 전환과 동일). 인증된 사용자는 목적지(`/app`)로 직행이 정석.
  실패 시엔 ProtectedRoute가 알아서 `/login`으로 보냄.

## 3. 최종 구조 (프론트 `src/`)
```
src/
├─ auth/            # 재사용 인증 모듈 (배럴 import)
│  ├─ client.ts         createAuthClient({baseUrl}) — login/signup/logout/tryRefresh/fetchMe/apiFetch
│  ├─ authClient.ts     기본 인스턴스 (VITE_API_BASE)
│  ├─ AuthContext.tsx   AuthProvider(client 주입가능) + useAuth
│  ├─ ProtectedRoute.tsx / GuestRoute.tsx   (fallback prop)
│  └─ index.ts
├─ notifications/   # 재사용 토스트 모듈 (NotificationProvider + useNotify)
└─ app/pages/       LoginPage(/login), SignUpPage(/register), NotFoundPage, RouteErrorPage
                    # OAuth 콜백 페이지 없음 — 백엔드가 /app 으로 직행
```

토큰 모델(직전 노트와 동일): **AT는 메모리만**, **RT는 백엔드 HttpOnly 쿠키**, 새로고침 시 silent refresh로 복원.
동시 refresh는 in-flight dedup으로 1요청 합침(StrictMode 이중 실행 + RT 재사용탐지 폭파 방지).
