# 🔐 인증/인가 학습 로드맵 (MOC)

> [!abstract] 한 문장
> 관통 프로젝트는 **PMS(+Wiki)**. 단계가 올라갈수록 **같은 앱을 마이그레이션**한다 — v1 세션 → v2 JWT → v3 Keycloak.

이 폴더의 지도. 각 단계의 디테일은 단계별 노트에서 본다. 학습은 "읽었다"가 아니라 **"실습 프로젝트가 돌아간다"**로 판정한다.

## 📍 진행 상황

- **0~7단계 완료.** 현재 위치: 8단계(Keycloak) 진입 직전 → MSA 트랙의 [[phase0-week-breakdown|Phase 0]]과 합류한다.
- 실습 프로젝트 (`C:\myStudy\` 하위):
  - 1단계 `sessionLogin` — Boot4 / Security7 / MySQL(Docker)
  - 3단계 `jwt_Login` — Boot4 / Thymeleaf, 라이브러리·Security 없이 JWT 직접 구현
  - 4단계 `tokenSecurity` — Boot4 / Redis(Docker), AT·RT 회전·재사용탐지·블랙리스트·HttpOnly 쿠키
  - 5단계 `springSecurity` — 4단계를 Spring Security 표준 부품으로 이관
  - 6·7단계 `oauth-oidc-login` — Google OIDC 로그인 + JWT 모듈 재사용

## 🪜 학습 원칙

1. **마이그레이션식 학습.** 새 앱을 만들지 않고 같은 도메인을 단계마다 더 나은 인증 방식으로 갈아끼운다.
2. **직접 짜는 양은 단계가 올라갈수록 줄어든다.** 3단계가 마지막으로 "밑바닥을 손으로" 짜는 단계 → 5단계는 표준 부품에 위임 → 8·9단계는 Keycloak이 발급·검증을 대신함.
3. **암기보다 설명.** 각 단계는 "코드를 외웠나"가 아니라 "왜 이렇게 동작하는지 말로 설명되나"로 끝낸다.
