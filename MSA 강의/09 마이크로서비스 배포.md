# 🚀 마이크로서비스 배포 (Deployment with Containers & Orchestrators)

## 1️⃣ Docker and Kubernetes Architecture
- **Docker** — 각 서비스를 **컨테이너**로 패키징 (실행 환경째로 묶음 → 어디서나 동일하게 실행)
- **Kubernetes(K8s)** — 수많은 컨테이너를 **자동으로 배치·확장·복구**하는 오케스트레이터
- 각 서비스(Product, Cart, User…)가 Docker 컨테이너로 동작하고, 전체를 Kubernetes가 감쌈

## 2️⃣ Sidecar Pattern & Service Mesh Pattern
- **Sidecar Pattern** — 메인 서비스 옆에 **보조 컨테이너**를 붙여 공통 기능(로깅, 모니터링, 통신 등)을 분리 처리
- **Service Mesh Pattern** — 사이드카들을 모아 **서비스 간 통신을 전담하는 인프라 계층** 구성 (예: Istio)
  - 비즈니스 로직과 통신·보안·관측 로직을 분리

## 3️⃣ DevOps and CI/CD Pipelines
- **CI (Continuous Integration)** — 코드 변경을 자동으로 빌드·테스트·통합
- **CD (Continuous Delivery/Deployment)** — 자동으로 배포까지 연결
- → 빠르고 안정적인 반복 배포 가능

## 4️⃣ Deployment Strategies (배포 전략)

| 전략 | 방식 | 특징 |
|---|---|---|
| **Blue-Green** | 구버전(Blue)·신버전(Green) 환경을 동시에 두고 한 번에 전환 | 빠른 롤백 가능 |
| **Rolling** | 인스턴스를 조금씩 순차적으로 교체 | 무중단, 점진적 |
| **Canary** | 일부 사용자(소량)에게 먼저 신버전 노출 후 점진 확대 | 위험 최소화 |
| **A/B** | 두 버전을 동시 운영하며 비교 | 기능·성과 테스트 |

## 5️⃣ Infrastructure as Code (IaC)
- **인프라를 코드로 정의·관리** (예: Terraform, Ansible)
- 서버·네트워크 설정을 수동이 아닌 코드로 → 재현 가능, 버전 관리, 자동화

## 🗺️ 다이어그램 한눈에

```text
각 서비스 = Docker 컨테이너 (Product, Cart, User, Order, Payment)
        │
        ▼
   Event Hub / Event Backbone
        │
   전체를 Kubernetes가 오케스트레이션
   (배치 · 확장 · 복구 자동화)
```

---

## ✅ 한 줄 정리
> 각 서비스를 **Docker 컨테이너**로 묶고 **Kubernetes**로 운영하며,
> **CI/CD 파이프라인**과 **배포 전략(Blue-Green/Rolling/Canary/A-B)**, **IaC**로
> 안전하고 자동화된 배포를 실현한다.
