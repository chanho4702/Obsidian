## 📌 1. 기능 요구사항 (Functional Requirements)

### 🏷️ 상품 (Product)
- [ ] 상품 추가, 수정, 삭제
- [ ] 카테고리 관리
- [ ] 상품 검색
- [ ] 필터링 및 정렬

### 👤 사용자 (User)
- [ ] 회원가입 / 로그인 / 로그아웃
- [ ] 프로필 관리
- [ ] 소셜 로그인 지원

### 🛍️ 장바구니 & 주문 / 결제 (Cart & Order / Payment)
- [ ] 장바구니 추가, 삭제
- [ ] 주문 생성
- [ ] 결제 처리 (신용카드, 가상계좌 등)
- [ ] 배송 추적 기능
- [ ] 결제 성공 / 실패 처리
- [ ] 결제 결과에 따른 주문 상태 업데이트

### 📦 재고 (Inventory)
- [ ] 상품 재고 확인 및 업데이트
- [ ] 인기 상품 재고 부족 시 알림

### ⭐ 리뷰 (Review)
- [ ] 사용자 리뷰 및 평점 작성
- [ ] 리뷰 신고 기능
- [ ] 부적절한 리뷰 필터링

### 🔔 알림 & 고객지원 (Notification & Support)
- [ ] 이메일 / 푸시 알림 (주문 상태 변경, 프로모션)
- [ ] FAQ 제공
- [ ] 고객 문의
- [ ] 실시간 채팅 지원

### 🛠️ 관리자 (Admin)
- [ ] 주문 / 결제 / 사용자 관리 대시보드
- [ ] 매출 통계 및 트렌드 분석

> 💡 **참고** — 위 도메인(상품·사용자·장바구니·주문·결제·재고…)이 그대로 마이크로서비스 분해 시
> 서비스 경계(Service Boundary)의 후보가 됩니다. → [[04 마이크로서비스 아키텍처 설계 영역]]

## ⚙️ 2. 비기능 요구사항 (Non-functional Requirements)

| 구분 | 항목 | 세부 내용 |
|---|---|---|
| **확장성** | Scalability | 동시 사용자 증가 시 성능 유지 / 트래픽 급증 대응 |
| **보안** | Security | 사용자 데이터 보호(암호화, 인증·인가) / 결제 정보 보호 |
| **가용성** | Availability | 장애 시 빠른 복구(Failover) / 데이터 백업·복원 |
| **응답 속도** | Performance & Latency | 페이지 로딩 3초 이내 / DB 쿼리 최적화 |
| **테스트 가능성** | Testability | 단위·통합 테스트 설계 / 자동화된 배포 파이프라인 |

---

### 📋 비기능 요구사항 상세

#### 🔼 확장성 (Scalability)
- 동시 사용자 증가 시에도 성능 유지
- 트래픽 급증 상황 대응
- → 수평 확장(Horizontal Scaling)·Load Balancer로 대응 ([[03 설계 원칙과 아키텍처별 설계 방법|03]])

#### 🔐 보안 (Security)
- 사용자 데이터 보호 (암호화, 인증 및 인가)
- 결제 정보 보호
- → JWT/OAuth2 인증, HTTPS, API Gateway Offloading으로 적용 ([[05 마이크로서비스 통신|05]])

#### 🟢 가용성 (Availability)
- 장애 발생 시 빠른 복구(Failover) 지원
- 데이터 백업 및 복원 기능
- → Circuit Breaker·Bulkhead 등 회복성 패턴과 연결 ([[08 마이크로서비스 회복성·관측성·모니터링|08]])

#### ⚡ 응답 속도 (Performance & Latency)
- 페이지 로딩 시간 최적화 (3초 이내)
- 데이터베이스 쿼리 최적화
- → 캐싱(Redis)·CQRS 읽기 최적화와 연결 ([[06 마이크로서비스 데이터 관리|06]])

#### 🧪 테스트 가능성 (Testability)
- 단위 테스트 및 통합 테스트 설계
- 자동화된 배포 파이프라인 지원
- → CI/CD 파이프라인으로 실현 ([[09 마이크로서비스 배포|09]])
